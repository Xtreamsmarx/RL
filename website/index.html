Algorithm Coverage

Dynamic Programming (Exact Methods)

Policy Iteration

Value-based | On-policy

Alternates between policy evaluation and improvement until convergence. Requires model knowledge.

✓ Exact ✓ Guaranteed Convergence ✗ Requires Model

Value Iteration

Value-based | Off-policy

Combines policy evaluation and improvement in single step. Computationally efficient exact method.

✓ Exact ✓ Efficient ✗ Requires Model

Monte Carlo (Model-Free Averaging)

MC Exploring-Starts

Value-based | On-policy Control

Random episode starts; averages full episode returns. Unbiased but high variance.

✓ Model-Free ✓ Unbiased ✗ Requires Exploring Starts

MC ε-Greedy

Value-based | On-policy Control

Epsilon-greedy exploration; practical alternative to exploring starts.

✓ Model-Free ✓ Practical ✓ ε-Greedy

Temporal Difference (Bootstrapping)

TD(0) Prediction

Value-based | On-policy

Single-step bootstrapping. Foundation for all TD methods.

✓ Model-Free ✓ Bootstrap ✓ Low Variance

n-Step TD Control

Value-based | On-policy

Multi-step returns; balance between TD(0) bias and MC variance.

✓ Tunable Bias-Variance ✓ Bootstrap ✗ Needs Lookahead

TD(λ) Forward-View

Value-based | On-policy

Weighted average of n-step returns via λ parameter.

✓ Elegant Theory ✓ λ-Returns ✗ Offline

TD(λ) Backward-View

Value-based | On-policy

Eligibility traces; equivalent to forward-view but online and efficient.

✓ Online ✓ Efficient ✓ Traces

Q-Learning

Value-based | Off-policy Control

Off-policy learning; learns optimal policy while following exploratory behavior policy.

✓ Off-Policy ✓ Optimal ✓ Industry Standard

SARSA (On-Policy Action-Value)

SARSA(0)

Action-Value | On-policy

Single-step on-policy control. Uses actual next action (not greedy).

✓ On-Policy ✓ Safe ✓ Conservative

n-Step SARSA

Action-Value | On-policy

Multi-step on-policy control; practical exploration-exploitation balance.

✓ On-Policy ✓ Multi-Step ✓ Tunable

SARSA(λ) Forward-View

Action-Value | On-policy

On-policy eligibility traces; λ-returns for action-values.

✓ On-Policy ✓ λ-Returns ✓ Traces

SARSA(λ) Backward-View

Action-Value | On-policy

Efficient online on-policy traces; practical replacement/accumulating trace options.

✓ Online ✓ Efficient ✓ Flexible

Deep Reinforcement Learning

Deep Q-Network (DQN)

Neural Network | Value-based | Off-policy Control

Neural network approximation of Q-values with experience replay and target networks. Stabilizes deep Q-learning with two key innovations:

Online Network: Updated every step via gradient descent
Target Network: Separate network for target estimation, updated periodically
Experience Replay: Samples transitions from buffer to break correlation
ε-Decay Exploration: Starts high, decays over episodes
✓ Discrete Actions ✓ Off-Policy ✓ Stable Training ✓ Deep RL Baseline

V3 Strategic Decisions

✅ What Was Implemented

Classical RL (14 Algorithms)

Complete pedagogical coverage:

  • DP: Policy Iteration, Value Iteration (exact methods)
  • MC: Exploring-starts, ε-greedy (model-free averaging)
  • TD Prediction: TD(0), TD(λ) forward-view (bootstrapping)
  • TD Control: n-step TD, TD(λ) backward-view, Q-learning (multi-step methods)
  • SARSA: SARSA(0), n-step SARSA, SARSA(λ) forward/backward (on-policy)

Deep RL (1 Algorithm)

DQN: Discrete action Q-learning with replay buffer, target networks, and ε-decay exploration

Environment

FrozenLake-v1: 4×4 grid, discrete state/action, deterministic (ideal for learning progression)

❌ What Was NOT Implemented (& Why)

❌ Swarm Coordination & Multi-Agent RL

Reason: Single-agent environment. Coordination mechanisms are orthogonal to capstone scope.

❌ Continuous Control (PPO, TRPO, SAC, DDPG, TD3)

Reason: Discrete action space. Continuous RL overkill for FrozenLake; DQN is architecturally superior.

❌ Model-Based Methods (MCTS, Planning)

Reason: Fully observable, deterministic environment. No need for learned model; model-free dominates.

❌ Hierarchical RL / Options Framework

Reason: 16-state space lacks natural task hierarchy. Temporal abstraction unjustified.

❌ Rainbow DQN (6-Component Enhancement)

Reason: Each component (prioritized replay, dueling, distributional) adds complexity exceeding pedagogical return for FrozenLake's simplicity.

❌ Bayesian Hyperparameter Tuning

Reason: Simple hyperparameters (α, γ, λ, ε) have clear ranges. Brute-force grid search faster than Bayesian optimization.

❌ Meta-Learning & Few-Shot Adaptation

Reason: Single-task environment. Meta-learning applies to multi-task distributions; not applicable here.

🎯 Environmental Fit Rationale

Observability Fully Observable No POMDP complexity; state == complete information
Stochasticity Deterministic Option Isolates algorithm behavior; eliminates environmental noise confounds
